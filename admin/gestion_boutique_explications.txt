Le fichier Gestion boutique effectue plusieurs traitements : 

0 Accessibilite / Restriction
Si l'utilisateur connecté (en session) n'est pas admin, il est redirigé.

//-------------------
1 Affichage de tous les produits !! 
1.1 Requete SELECT
2.2 Boucles pour afficher toutes les infos sous forme de tableau
	2.2.1 On écrit pas l'image mais on l'affiche dans une balise IMG
2.3 Pour chaque enregistrement on ajoute un lien pour modifier, et un lien pour supprimer avec l'id_produit dans l'url. 

//------------------------------------
2 Ajout d'un produit
2.1 Affichage du formulaire pour ajouter (ou modifier) un produit
2.2 Gestion de l'image
	2.2.1 Vérifier que l'image n'est pas vide
	2.2.2 Modifier le nom de l'image (éviter les collisions)
	2.2.3 Reconstituer le chemin exacte (absolu) de l'image à insérer 
	2.2.4 Copier/coller l'image depuis son emplacement temporaire ($_FILE['photo']['tmp_name']) vers son emplacement définitif (fonction copy()).
2.3 Requete pour insérer

//-------------------------------------
3 Suppression d'un produit
3.0 On récupérer la demande d'action de suppression via le GET (dans l'URL)
3.1 On récupérer l'id_produit du produit à supprimer également via le GET (dans l'URL)
3.2 Avant de supprimer un produit on supprime la ou les photo(s) qui correspond(ent) à ce produit (fonction unlink()). 
3.3 Requete DELETE pour supprimer le produit en question. 

//-------------------------------------
4 Modifier un produit
4.0 On récupérer la demande d'action de modification via le GET (dans l'URL)
4.1 Afficher le formulaire
4.2 On récupère l'id_produit du produit à modifier via le GET (dans l'URL)
4.3 Récupérétion et Affichage les infos du produit dans le formulaire
	4.2.1 Champs file : Ca ne fonctionne pas. On peut pas utiliser 'value'. On va afficher la photo dans une balise IMG. 
	4.2.2 Créer un champs 'hidden' (input caché) dans lequel on va metre en value le nom de la photo du produit à modifier. 
	4.2.3 Avant de modifier le produit on vérifiera si un champs photo actuelle existe et donc on récupère le nom de cette photo. 
4.4 Créer un champs 'hidden', l'id du produit en cours de modification. L'id sera donc récupérable via le POST. 
4.5 Requete REPLACE ou UPDATE selon qu'il s'agisse d'une action d'ajout ou de modification.
